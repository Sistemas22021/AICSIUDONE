# Factibilidad Técnica

## Descripción

La factibilidad técnica evalúa si el sistema de gestión de incidentes y patrullaje policial puede desarrollarse utilizando tecnologías disponibles, considerando la capacidad técnica del equipo, la existencia de soluciones similares y los recursos necesarios para su implementación y despliegue.

## Tecnologías a utilizar

Para el desarrollo del sistema se propone el uso de las siguientes tecnologías:

* **Backend:** Java con Spring Boot, para la lógica del negocio y gestión de servicios.
* **Frontend:** TypeScript con React, para la construcción de la interfaz del sistema.
* **Base de datos:** PostgreSQL, para el almacenamiento de incidentes, patrullas, asignaciones e historial operativo.
* **Comunicación en tiempo real:** WebSockets implementados mediante NestJS, para la sincronización de estados e información operativa.
* **Geolocalización:** API de Google Maps para registro georreferenciado y visualización de incidentes.
* **Automatización y procesamiento:** Python para scripts auxiliares y procesamiento complementario.

## Software de desarrollo

Para el desarrollo del sistema se requiere el uso de las siguientes herramientas:

* IntelliJ IDEA o Visual Studio Code.
* Java JDK.
* Node.js y npm.
* PostgreSQL.
* Git y GitHub para control de versiones.
* Postman para pruebas de servicios.
* Docker (opcional para pruebas y despliegue).

## Hardware de desarrollo

Requisitos mínimos para el ambiente de desarrollo:

* Procesador Intel i5 o equivalente.
* Memoria RAM mínima de 8 GB.
* 10 GB libres de almacenamiento.
* Conexión estable a internet.

Estos recursos permiten desarrollar, probar e integrar adecuadamente la solución propuesta.

## Software necesario para lanzamiento y despliegue

Para la puesta en marcha del sistema se requiere:

* Servidor o servicio de hosting para desplegar la aplicación.
* Servidor de aplicaciones compatible con Spring Boot.
* Instancia de PostgreSQL para operación del sistema.
* Navegador web actualizado para acceso de usuarios.
* Servicios habilitados de Google Maps API.
* Servicios WebSocket para comunicación en tiempo real.

## Capacidad técnica del equipo

El desarrollo del sistema es viable desde el punto de vista técnico, debido a que el equipo posee conocimientos en programación web, desarrollo frontend y backend, manejo de bases de datos e integración de tecnologías requeridas. Adicionalmente, las tecnologías seleccionadas cuentan con comunidades activas, soporte y recursos de aprendizaje que facilitan su implementación.

## Existencia de soluciones similares

Existen soluciones similares en entornos reales que gestionan incidentes y despacho de unidades en tiempo real, lo que valida la factibilidad técnica de la propuesta. Asimismo, existen referencias tecnológicas y proyectos que implementan comunicación en tiempo real, visualización geográfica y gestión de eventos utilizando herramientas equivalentes.

## Conclusión de Factibilidad Técnica

La factibilidad técnica del proyecto es alta y viable, debido a que las tecnologías seleccionadas son compatibles entre sí, los recursos de hardware y software son suficientes, y el equipo posee capacidades para desarrollar e implementar la solución propuesta. Además, la existencia de soluciones similares y la disponibilidad de herramientas y soporte técnico respaldan la ejecución del proyecto.
