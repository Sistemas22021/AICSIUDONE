# Factibilidad Operativa — Equipo Verde

## 1. Usuarios del sistema
- Perito forense: registra evidencias, carga imagenes de comparacion y documenta resultados tecnicos.
- Investigador: consulta evidencias, revisa comparaciones y correlaciona hallazgos con casos activos.
- Supervisor: valida resultados, autoriza cierres de analisis y audita trazabilidad del proceso.

## 2. Operadores del sistema
- Administrador funcional: configura catalogos, parametros operativos y permisos por rol.
- Soporte tecnico: atiende incidencias de uso, disponibilidad y rendimiento.
- Responsable de datos/auditoria: verifica integridad de registros y cumplimiento de protocolos.

## 3. Condiciones operativas necesarias
- Definicion de flujo estandar: registro -> analisis -> validacion -> cierre.
- Manual de roles y responsabilidades por tipo de usuario.
- Criterios de calidad de datos para cada evidencia registrada.
- Politica de control de acceso y trazabilidad obligatoria de acciones.

## 4. Estrategias para operacion adecuada
- Capacitacion inicial por perfil (perito, investigador, supervisor) con ejercicios de casos reales.
- Guias rapidas de uso para registro de evidencia y comparacion balistica.
- Plan de adopcion por etapas: piloto con casos controlados y expansion progresiva.
- Mesa de ayuda con tiempos de respuesta definidos para incidencias criticas y no criticas.
- Rutina de revision semanal de bitacoras para detectar errores recurrentes y oportunidades de mejora.

## 5. Riesgos operativos y mitigaciones
- Riesgo: resistencia al cambio por parte de usuarios de procesos manuales.  
  Mitigacion: acompanamiento en sitio, capacitacion y retroalimentacion continua.
- Riesgo: registros incompletos o inconsistentes.  
  Mitigacion: validaciones obligatorias en formularios y checklist de calidad.
- Riesgo: uso inadecuado de permisos.  
  Mitigacion: segregacion de funciones, auditoria y revisiones periodicas de acceso.

## 6. Conclusion operativa
La implementacion es operativamente factible si se ejecuta una estrategia formal de capacitacion, adopcion gradual y soporte continuo, respaldada por controles de calidad de datos y gobierno de roles.
